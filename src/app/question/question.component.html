Angular Interview Questions and Answers
1. What is Angular? Explain its main features.
Angular is a TypeScript-based front-end web application framework developed by Google. It allows developers to build single-page applications (SPAs) with a clean structure and reusable components.
Main Features: - Component-based architecture - Dependency injection - Two-way data binding - Directives for DOM manipulation - RxJS for reactive programming - TypeScript support
2. Explain the difference between AngularJS and Angular.
AngularJS: - Based on JavaScript - Uses MVC architecture - Supports two-way data binding but with a digest cycle - Released in 2010
Angular (2+): - Based on TypeScript - Component-based architecture - Enhanced performance with Ahead-of-Time (AOT) compilation - Supports hierarchical dependency injection - Released from 2016 onwards
3. How does Angular handle dependency injection?
Angular provides a built-in dependency injection (DI) framework that allows developers to define and manage dependencies for components, services, and other classes.
How it works: - Dependencies are declared in the constructor of a class. - Angular injects instances of these dependencies at runtime, ensuring loose coupling between components and services. - The hierarchical injector system allows scoped services for modules and components.
4. What are lifecycle hooks in Angular?
Lifecycle hooks are methods that Angular calls at specific points in a component or directive's lifecycle.
Common Lifecycle Hooks: - ngOnInit(): Called once the component is initialized. - ngOnChanges(): Called whenever input properties change. - ngOnDestroy(): Called before the component is destroyed. - ngAfterViewInit(): Called after the component's view is initialized. 5. Explain Change Detection in Angular.
Change detection in Angular is the process of tracking changes in the state of an application and updating the DOM accordingly.
How it works: - Angular uses a tree of components to detect changes. - Each component has its own change detector, which checks bindings and updates the DOM. - The Zone.js library is used to detect asynchronous changes like HTTP requests or user input. - Developers can optimize change detection using OnPush change detection strategy or manually triggering detection using ChangeDetectorRef.
6. How do you optimize an Angular application?
Optimizing an Angular application involves various strategies: - Use Ahead-of-Time (AOT) compilation to reduce the size of the application bundle. - Implement lazy loading to load modules only when needed. - Use OnPush change detection for components with immutable data. - Minimize the use of watchers and unnecessary bindings. - Compress assets like images and CSS files. - Use Angular Universal for server-side rendering (SSR). - Leverage performance tools like Lighthouse to identify bottlenecks.
